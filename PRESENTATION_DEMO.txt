Every year, thousands of pitch decks land in investors' inboxes. The dirty truth:

  • 73% of pitch decks contain exaggerated TAM (Total Addressable Market) claims
  • Founders routinely cite data that is 6–18 months stale
  • "AI-powered" is the new "blockchain-powered" — buzzwords replace substance
  • Financial projections go completely unchallenged until the term sheet

The industry's current "solution" is to paste the deck into ChatGPT and ask
"Is this legit?" ChatGPT gives you an OPINION. BluffBuster gives you EVIDENCE
WITH RECEIPTS.

  SECTION 2 — WHAT IS BLUFFBUSTER?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

BluffBuster is a FORENSIC AI SYSTEM that:

  1. EXTRACTS every factual, quantitative, and comparative claim from a pitch
     deck PDF automatically.

  2. SEARCHES the live web (DuckDuckGo + Jina AI deep-scraping) for
     corroborating or contradicting evidence in real time.

  3. VERIFIES each claim with one of four machine verdicts:
       → VERIFIED       (claim is well-supported)
       → EXAGGERATED    (technically true but overstated)
       → FALSE          (contradicted by live evidence)
       → UNVERIFIABLE   (no reliable evidence either way)

  4. DEBATES the most contentious claims in a live adversarial courtroom:
       → Defender Agent  argues in the founder's favor
       → Prosecutor Agent cites contradicting data and tears the claim apart

  5. VISUALIZES every finding on a per-page Truth Heatmap with a weighted
     overall Trust Score (0.0 — 1.0).

  ALL OF THIS IN UNDER 90 SECONDS.

── DUAL-MODE ANALYSIS ────────────────────────────────────────────────────────

  FOUNDER MODE 🟢 (Self-Defense)
  --------------------------------
  Designed for founders who want to stress-test their own deck BEFORE
  investors do.

  Features:
  • First Impression Analyzer   — Scores opening 3 slides for hook strength,
                                   problem clarity, and solution comprehension.
                                   Rewrites weak slides on the spot.
  • Claim Repair Feed           — For every FALSE or EXAGGERATED claim, the AI
                                   generates a defensible, rewritten version
                                   backed by real evidence.
  • Financial Stress Test       — Benchmarks projections against real industry
                                   medians. Flags "100% YoY for 5 years" as
                                   implausible vs. SaaS median of ~30%.
  • Competitor Blind Spot Detector — Finds competitors you forgot to mention.
                                     Before investors find them for you.
  • Adversarial Debate Engine   — Tests your deck's weakest claims in a
                                   Defender vs. Prosecutor courtroom.
  • Truth Heatmap               — Color-coded per-page visualization of claim
                                   integrity. 🟢 Green | 🟡 Amber | 🔴 Red
  • Temporal Claim Decay        — Shows which claims have a "freshness score"
                                   and are about to expire due to market shifts.

  VC MODE 🟡 (Due Diligence Weaponized)
  ----------------------------------------
  Designed for investors who want forensic-grade due diligence in under 90s.

  Features:
  • Red Flag Scorer             — Weighted severity scoring: Critical / Major /
                                   Minor. Prioritizes which claims need follow-up
                                   questions.
  • Investor Fingerprint        — Runs the analysis through the specific lens
                                   of a VC firm's thesis (see below).
  • Domain Credibility Check    — Cross-references founding team claims about
                                   domain expertise with their actual content.
  • Competitor Mapper           — Structured competitive intelligence: funding
                                   rounds, trajectories, moat comparison.
  • Adversarial Debate Engine   — Same courtroom mode as Founder, but from the
                                   investor's critical perspective.
  • Truth Heatmap + Trust Score — Same forensic visualization.
  • Temporal Claim Decay        — Flags stale market claims with expiry dates.

── INVESTOR FINGERPRINT SYSTEM (UNIQUE TO VC MODE) ──────────────────────────

  Run the same deck through 5 different VC lenses. Each persona biases the
  analysis to that firm's documented investment thesis:

  • Standard       — Balanced forensic audit. Trust no one without proof.
  • Sequoia Capital — Market dominance obsessed. Penalizes small TAMs brutally.
  • Y Combinator   — Velocity & founder-market fit. Hates bloated tech stacks.
  • Tiger Global   — Aggressive scale. Obsessed with YoY revenue & GMV.
  • Benchmark      — Unit economics. Penalizes high CAC & marketing-led growth.

  A deck rejected by Sequoia's lens might pass YC's. BluffBuster shows you why.

── ADVERSARIAL DEBATE ENGINE ─────────────────────────────────────────────────

  For the top 3 most contentious claims (FALSE or EXAGGERATED), two AI agents
  stage a live courtroom argument:

  • Defender 🟢  — Argues the founder's case. Finds charitable interpretations.
  • Prosecutor 🔴 — Cites evidence. Aggressively dismantles the claim.

  Users can INTERVENE via chat — submit your own argument and a Judge Agent
  issues an official ruling: SUSTAINED / OVERRULED / PARTIALLY_SUSTAINED.

── TRUTH HEATMAP ─────────────────────────────────────────────────────────────

  Every page of the deck gets a score:
  • > 0.7 = 🟢 Green (trustworthy)
  • 0.3–0.7 = 🟡 Amber (caution)
  • < 0.3 = 🔴 Red (high-risk page)

  Business-critical categories (TAM, growth, financial, competitor) are
  weighted 3x more than soft categories (team, general).

  Result: A single number that captures the deck's forensic integrity.

── TEMPORAL CLAIM DECAY ──────────────────────────────────────────────────────

  "We are the only player in X" — how long does this claim stay true?

  BluffBuster fetches LIVE data on recent funding rounds and new market entrants,
  then assigns each claim a Freshness Score (0.0 – 1.0) and an estimated
  claim expiry date. Investors know which claims to verify NOW.

  SECTION 4 — HOW IT WORKS (PIPELINE WALKTHROUGH)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  [1] User uploads a PDF pitch deck (drag & drop, max 20MB)

  [2] Backend extracts text from every page using pdfplumber (local, ~2s)

  [3] LLM extracts all factual/quantitative claims — categorized as:
      TAM | Growth | Competitor | Financial | Technical | Team | General

  [4] For each claim, a web evidence pipeline runs:
      → DuckDuckGo search (5 results per claim)
      → Top result deep-scraped via Jina AI Reader (full-page markdown)
      → Keyword-relevant context extracted (±15 lines)

  [5] ALL claims + ALL evidence bundled into ONE batch LLM call.
      Uses [ID: CX] tags for deterministic 1:1 claim-to-verdict mapping.
      → Output: verdict + confidence + reasoning + source URLs

  [6] Mode-specific agents run in parallel:
      → Founder: First Impression + Claim Repair + Financial Stress + Blind Spots
      → VC: Red Flag Scoring + Domain Credibility + Competitor Mapping

  [7] Temporal Decay agent fetches live market data, assigns freshness scores.

  [8] Adversarial Debate — single batch LLM call generates both sides
      simultaneously for the top 3 contentious claims.

  [9] Truth Heatmap computed (no LLM needed — pure math, <1s)

  [10] Dashboard renders all results live via React + WebSocket.

  TOTAL: ~90 seconds end-to-end for a 15-slide deck.

BACKEND
─────────────────────────────────────────────────────────────────────────────
  Language:     Python 3.12
  API Server:   FastAPI (REST + WebSocket)
  Agent Graph:  LangGraph (StateGraph-based multi-agent orchestration)
  PDF Parsing:  pdfplumber (local extraction, no API cost)
  Web Search:   DuckDuckGo Search API (free, no rate limits)
  Web Scraping: Jina AI Reader (converts any URL to clean markdown)
  Embeddings:   Sentence Transformers — all-MiniLM-L6-v2 (runs LOCAL)
  Vector DB:    ChromaDB (in-memory, semantic claim similarity)
  Database:     SQLite (zero-config session persistence)
  Validation:   Pydantic v2
  LLM Layer:    OpenAI-compatible client (works with any provider:
                AICredits / OpenRouter / Groq / OpenAI)

  LLM RESILIENCE:
  • Exponential backoff — up to 8 retries on 429 rate-limit errors
  • JSON-mode fallback — retries without response_format if unsupported
  • Multi-strategy JSON parser: direct → markdown block → brace match →
    bracket match → positional fallback

FRONTEND
─────────────────────────────────────────────────────────────────────────────
  Framework:    React 19 + TypeScript 5.8
  Build Tool:   Vite 6.2
  Styling:      TailwindCSS 4.1 (utility-first)
  Animations:   Framer Motion 12.x (declarative UI transitions)
                GSAP 3.15 (scroll-triggered animations, horizontal scroll)
  Smooth Scroll: Lenis 1.3
  State:        Zustand 5.0 (lightweight global store)
  Charts:       Recharts 3.8 (data visualization)
  Routing:      React Router 7.14

  UI DESIGN SYSTEM:
  • Retro Arcade × Dark Forensics aesthetic
  • 4-color palette: Red (#FF3D3D), Amber (#FF9500), Green (#00C896),
    Black (#0A0A0F)
  • Fonts: Press Start 2P (pixel art headings), Bebas Neue (section titles)
  • Glassmorphism cards with backdrop blur + neon glow variants
  • CRT scanline overlay for retro authenticity
  • Mouse-tracking spotlight effect on claim cards
  • Globally animated gradient blobs (ambient atmosphere)

INFRASTRUCTURE
─────────────────────────────────────────────────────────────────────────────
  • Provider-agnostic LLM layer — swap any OpenAI-compatible API via .env
  • Disk-based search cache (MD5 hash) — repeated analyses skip web search
  • Session-based architecture — stateless frontend, all state in SQLite
  • WebSocket streaming — live progress updates during analysis
